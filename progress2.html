In the first course, Syntax, actions were mentioned already. Actions are operations which the object (the main character) can execute. Using an action has the following syntax:

object.nameOfAction( parameter1, parameter2 );

One of the examples we saw before was i.moveTo("John"); . moveTo("John") is the action in this example. Let's take a closer look at the structure of this action.

moveTo is the name of this action. The name of an action is written without spaces or interpunction. To ensure the name is still readable, we distinguish different words by capitalizing the first letter of a new word. This practise is called camel case, often written like camelCase.

As discussed in the first course as well, every action should be followed by parentheses (). The reason for this is that some actions require additional information in order to be executed properly. This information is called the parameter of an action. Following up on the example of i.moveTo("John");, "John" is the parameter. You are telling the computer that the i object needs to move somewhere, so you need to specify where exactly you want the object to move.

Note 1: It is not possible, with certain exceptions, to connect multiple actions on one line. For example, i.moveTo("John").greet(); would result in the computer returning you an error message.
